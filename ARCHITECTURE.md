# Personal System Architecture

> 三层架构：数据层（Brain OS） + 方法层（工程技能） + 执行层（日常助理）。
> 本仓库是**数据层**的框架骨架。

## 数据层：Brain OS（本仓库）

**职责：** 存储你的所有知识、身份、人脉、目标、记忆。
**入口：** `CLAUDE.md` → 路由到具体模块。

```
.
├── CLAUDE.md          ← 总入口（规则 + 路由表）
├── AGENT.md           ← 核心铁律 + 决策表
├── ARCHITECTURE.md    ← 本文件
│
├── identity/          ← 【身份】单一真相源
├── memory/            ← 【记忆】fact 临时池 + 工作日志
├── operations/        ← 【运营】目标和进度
├── content/           ← 【内容创作】选题→研究→草稿→发布
├── knowledge/         ← 【知识库】书签 + 学习笔记
├── network/           ← 【人脉】联系人 + 互动
├── personal-brand/    ← 【个人品牌】声音指南
├── brand/             ← 【品牌资产】
├── templates/         ← 【模板】
├── skills/            ← 【技能定义】
└── automation/        ← 【自动化】周回顾、人脉维护
```

**数据流：**
```
任何对话 → 抽取 fact → fact.jsonl（临时池）
                          │
                    /整理记忆 触发
                          │
             ┌────────────┼────────────┐
             ▼            ▼            ▼
        identity/    operations/   knowledge/  ...（按 module_hint 路由）
```

## 方法层：工程技能（可选）

软件工程方法论，指导「怎么做」——brainstorming、writing-plans、test-driven-development、
systematic-debugging、code-review、verification 等。可由你的 AI 编程助手通过插件/技能机制接入。
项目级偏好记忆与本数据层分离，互不可见。

## 执行层：日常助理（可选）

日常助理、文件处理、定时任务、多渠道通讯。定期读取本数据层
（goals / fact / contacts / progress）做巡检和提醒。身份信息以本数据层
`identity/me.md` 为单一真相源，助理侧的 profile 仅为同步副本。

## 记忆系统分工

| 系统 | 记什么 |
|------|--------|
| **Brain OS `fact.jsonl`** | 人生记忆：学到的知识、做的决策、遇到的人、项目进展、个人想法 |
| **编程助手项目记忆** | 项目级工作偏好（如「用中文回复」「先跑测试再提交」） |
| **助理对话记忆** | 助理自己的对话记录和语义索引 |

**原则：** 身份信息单一真相源 = `identity/me.md`；三系统记忆互相独立，不假设互相可读。

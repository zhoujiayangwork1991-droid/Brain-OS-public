# 技能：整理记忆

**触发词**：`/整理记忆`

## 目标

读取 `memory/fact.jsonl` 中的临时事实，将每条 fact 归纳到对应模块的数据文件中，然后从 `fact.jsonl` 删除已归纳的条目。

---

## 执行步骤

### 第一步：读取 fact.jsonl

读取 `memory/fact.jsonl` 的全部内容。若文件为空或不存在，告知 JY "没有待整理的记忆" 并停止。

### 第二步：按模块分组

将每条 fact 按 `module_hint` 字段归类，整理成如下结构：

```
[模块名]
- fact 内容（日期）
- ...
```

向 JY 展示分组结果，并逐组确认是否归纳（可整体一次确认）。

### 第三步：写入对应模块

根据 `module_hint` 将 fact 追加到对应模块的 JSONL 文件：

| module_hint | 目标文件 |
|-------------|---------|
| personal-brand | `personal-brand/facts.jsonl` |
| content | `content/facts.jsonl` |
| knowledge | `knowledge/facts.jsonl` |
| network | `network/facts.jsonl` |
| operations | `operations/facts.jsonl` |
| identity | `identity/facts.jsonl` |
| brand | `brand/facts.jsonl` |
| memory | `memory/experiences.jsonl` |
| templates | `templates/facts.jsonl` |
| skills | `skills/facts.jsonl` |
| automation | `automation/facts.jsonl` |

写入格式（保持原字段，追加 `archived_at`）：
```json
{"date": "...", "fact": "...", "source": "...", "archived_at": "YYYY-MM-DD"}
```

**注意**：若目标模块文件不存在，直接创建并写入。

### 第四步：清理 fact.jsonl

将已归纳的条目从 `memory/fact.jsonl` 中删除，只保留未被归纳的条目（即 JY 拒绝归纳的部分）。

实现方式：用保留条目重写 `fact.jsonl`（这是唯一允许覆盖写 JSONL 的场景，因为是清理操作）。

### 第五步：汇报结果

输出简洁汇报：
```
整理完成：
- 归纳了 X 条记忆到 Y 个模块
- 保留了 Z 条待归纳
```

---

## 注意事项

- 若 JY 对某条 fact 的归属有异议，允许手动指定目标模块
- 若 fact 内容模糊无法归类，归入 `memory/experiences.jsonl`
- 整理过程中不修改 fact 的内容，只做搬运
